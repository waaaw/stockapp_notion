### 2. KIS(한국투자증권) 마일스톤 1 구현 — 읽기 전용 잔고 교차검증
- 사용자가 모의투자 신청 완료함에 따라 Part B 첫 마일스톤 코드 착수
- `config.py`: `kis_app_key`/`kis_app_secret`/`kis_account_no`/`kis_is_paper`(기본 true=모의투자)/
  `kis_base_url`(모의: openapivts…:29443, 실전: openapi…:9443) 추가. `kis_enabled`로 선택 기능 게이팅
- `kis_api.py` 신규:
  - 토큰 발급/디스크 캐싱(`.kis_token_cache.json`, 만료 1시간 전 재발급, 모의/실전 environment 구분)
  - `_request_with_retry`: 429/5xx 지수 백오프 (notion_api.call_with_retry와 동일 구조)
  - `fetch_account_balance()`: 주식잔고조회(TR: TTTC8434R/VTTC8434R), 연속조회 페이징 지원
  - `check_balance_against_notion()`: KIS 잔고 vs Notion 계산 결과 비교, 불일치 목록 반환
    (아무것도 덮어쓰지 않는 읽기 전용)
- `cli.py`: `kis-check-balance` 명령 추가, `daily-update`에 KIS 설정 시에만 교차검증 수행
  (실패해도 배치 계속 진행)
- `.env.example`에 KIS 키 4개 추가, `.gitignore`에 토큰 캐시 파일 추가, `requests` 의존성 추가
- 키 미설정 시 명확한 에러로 실패하는 것 확인. **실제 API 호출 검증은 사용자가 appkey/appsecret을
  발급받아 `.env`에 넣은 후 가능**

## 다음 단계
1. **사용자**: https://apiportal.koreainvestment.com/apiservice 에서 API 신청(모의투자용) →
   발급받은 `KIS_APP_KEY`/`KIS_APP_SECRET`/`KIS_ACCOUNT_NO`(모의투자 계좌번호 10자리)를 `.env`에 입력
2. `python -m stockapp_notion.cli kis-check-balance` 실행해 모의투자 잔고조회 동작 검증
   (TR ID/필드가 KIS 문서와 다르면 이 시점에 조정)
3. 검증 후 마일스톤 2: 체결내역 자동 임포트(`fetch_execution_history` + 중복방지) 진행
4. 모의투자로 전 과정 검증 후에만 `KIS_IS_PAPER=false`로 실계좌 전환
