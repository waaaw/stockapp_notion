### 2. Notion 실연동 및 버그 수정
- 사용자가 Notion Integration 생성(액세스 토큰 방식) 후 페이지 연결, `.env` 설정
- `scripts/setup_notion_databases.py`로 4개 DB(종목마스터/매매내역/포트폴리오요약/배당금) 생성
- **실연동 중 발견한 버그 2건**:
  1. `notion-client` 3.x부터 Notion API의 2025-09 "data source" 구조 변경으로
     `databases.query`가 제거되고 `data_sources.query`로 대체됨. DB 생성 페이로드도
     `properties` → `initial_data_source.properties`로 변경됨. 이 때문에 최초 생성한
     4개 DB에 커스텀 속성이 전혀 반영되지 않는 문제 발생 → `data_sources.update`로
     속성을 추가해 복구하고, 조회 코드 전체를 `data_sources.query` + 캐싱 헬퍼
     (`notion_api.resolve_data_source_id`)로 이관. `setup_notion_databases.py`도 새
     스키마로 재작성.
  2. `yfinance`의 `fast_info.get("last_price")`가 항상 `None`을 반환하는 버그
     → `fast_info.last_price` 속성 접근으로 수정.
  3. `requirements.txt`/`pyproject.toml`의 `notion-client>=2.2.1` 표기를 `>=3.0.0`으로
     상향(2.x에는애초 `data_sources` API 자체가 없어 동작 불가능했음)
- 실제 워크스페이스에서 `add-stock`/`add-transaction`/`update-prices`/`sync-portfolio`/
  `add-dividend`/`daily-update` 전체 흐름 검증 완료 (삼성전자 종목으로 테스트,
  평가손익 342.14% 등 계산 정확성 확인)
