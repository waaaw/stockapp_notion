# 주식 포트폴리오 관리 프로그램 — 사용 매뉴얼

이 문서는 이 프로그램을 **어떻게 쓰는지** 처음부터 끝까지 설명하는 사용 설명서입니다.
설치·연동 절차는 [README.md](../README.md)와 [라즈베리파이 설치 가이드](raspberry-pi-setup-guide.md)에,
직접 하나씩 검증하는 절차는 [TESTING.md](../TESTING.md)에 있습니다.

---

## 1. 이 프로그램은 무엇인가

개인 주식 포트폴리오(보유 종목·매매 내역·수익률·배당금)를 **Notion에 저장하고 자동으로
집계·시각화**하는 프로그램입니다.

- **저장소 = Notion** — 데이터는 전부 Notion 데이터베이스에 들어갑니다. 프로그램은 계산과 갱신을
  담당하고, 실제 데이터는 Notion 앱/웹 어디서나 볼 수 있습니다(모바일 포함).
- **시세 = yfinance** — 국내·미국·중국·홍콩 종목의 현재가와 환율을 자동으로 가져옵니다.
- **두 가지 사용 방식** — 브라우저 **웹 UI**(권장)와 터미널 **CLI**. 둘 다 같은 기능을 하며 섞어 써도 됩니다.
- **자동 갱신** — 매일 장 마감 후(16:00) 현재가와 포트폴리오를 자동 갱신하도록 스케줄러를 걸 수 있습니다.
- **(선택) 증권사 연동** — 한국투자증권(KIS) API로 실제 계좌 잔고와 내 기록을 대조할 수 있습니다.

## 2. 전체 구성

```
[시세/환율: yfinance] ─┐
                        ├─→ [이 프로그램 (계산·갱신)] ─→ [Notion DB 4개] ─→ 대시보드/모바일
[증권사: KIS API(선택)]─┘         ▲
                                  │
                        웹 UI(브라우저) 또는 CLI(터미널)로 입력·조작
```

**Notion 데이터베이스 4개:**
| DB | 역할 |
|---|---|
| 종목 마스터 | 종목명/코드/시장/섹터/통화/현재가 |
| 매매 내역 | 거래일자/종목/매수·매도/수량/단가/수수료/총거래금액 |
| 포트폴리오 요약 | 종목별 보유수량/평균단가/평가금액/평가손익/실현손익/총수익 (자동 계산됨) |
| 배당금 내역 | 지급일/종목/세전·세후 배당금 |

## 3. 꼭 알아야 할 핵심 개념

### 3-1. 세 가지 "수익"의 차이
- **평가손익 (미실현)**: 지금 보유 중인 종목의 `현재가 − 평균단가` 기준 손익. 아직 안 판 상태.
- **실현손익**: 이미 **매도한** 거래에서 확정된 손익. 매도 시점의 평균단가 기준으로 건별 계산되어 누적됩니다.
- **총수익(배당포함)**: `평가손익 + 누적 실현손익 + 누적 배당금`. "지금까지 이 종목으로 번 전체"에 가깝습니다.

> 예: 10주를 1,000원에 사서 4주를 1,200원에 팔면 → 실현손익 (1200−1000)×4 = 800원이 기록되고,
> 남은 6주는 평균단가 1,000원으로 계속 평가손익이 계산됩니다.

### 3-2. 평균단가 방식
**이동 가중평균** 방식입니다(FIFO 아님). 매도해도 평균단가는 유지되고 수량만 줄어듭니다.

### 3-3. 다중통화
국내(KRW)만 있으면 기존과 똑같이 동작합니다. 미국(USD)·중국(CNY)·홍콩(HKD) 종목을 추가하면:
- 대시보드 상단 통계는 **KRW로 환산한 총계**를 보여줍니다(환율은 yfinance 자동 조회).
- **통화별 소계** 표가 추가로 나타나 각 통화의 원화폐 기준 합계를 따로 볼 수 있습니다.

### 3-4. 계산은 언제 갱신되나
매매내역을 입력/수정/삭제하면 해당 종목의 포트폴리오 요약이 **즉시** 재계산됩니다.
현재가는 "현재가 갱신"을 실행할 때(또는 매일 자동 배치) 갱신됩니다.

## 4. 웹 UI 사용법 (권장)

### 4-1. 실행
```bash
python -m stockapp_notion.webapp
```
브라우저에서 **http://127.0.0.1:5000** 접속. (라즈베리파이/Docker로 상시 구동하는 방법은 README 참고)

### 4-2. 화면 구성
- **상단 통계 카드 4개**: 총 평가금액 / 평가손익(미실현) / 누적 실현손익 / 총수익(배당포함)
- **통화별 소계**: 해외 종목이 있을 때만 표시
- **보유 현황 표**: 종목별 수량·평균단가·평가금액·손익·수익률·실현손익·총수익
  (보유수량 0이 된 종목은 표에서 빠지지만, 실현손익은 합계에 계속 반영됩니다)
- **종목 비중 도넛 차트**
- **입력 폼(우측)**: 매매내역 / 종목 등록 / 배당금
- 각 종목의 **"내역" 링크** → 그 종목의 개별 거래 조회·수정·삭제 페이지

### 4-3. 자주 하는 작업
| 하고 싶은 것 | 방법 |
|---|---|
| 새 종목 추가 | "종목 등록" 폼 작성 → 등록 (등록 즉시 현재가도 조회됨). 시장을 고르면 통화가 자동 선택됨 |
| 매수/매도 기록 | "매매내역 입력" 폼에서 종목 선택 → 매수/매도, 수량·단가·날짜 입력 → 등록 (포트폴리오 자동 재계산) |
| 배당금 기록 | "배당금 입력" 폼 (세전/세후 금액) |
| 잘못 입력한 거래 수정/삭제 | 해당 종목 "내역" 링크 → 행에서 값 고치고 "수정" 또는 "삭제"(휴지통, 복구 가능) |
| 현재가만 갱신 | 상단 "현재가 갱신" 버튼 |
| 전체 다시 계산 | 상단 "포트폴리오 재계산" 버튼 |
| 한 번에 갱신 | 상단 "일일 갱신" 버튼 (현재가 + 재계산) |

### 4-4. 로그인 (외부 접속 시)
`.env`에 `WEB_USERNAME`/`WEB_PASSWORD`(+`FLASK_SECRET_KEY`)를 채우면 로그인 화면이 생깁니다.
비워두면 로컬 전용으로 인증 없이 동작합니다. 외부 노출 시 반드시 채우세요(README의 배포 절 참고).

## 5. CLI 사용법 (전체 명령어)

터미널에서 아래 형식으로 실행합니다. (가상환경 활성화 상태 기준)
```bash
python -m stockapp_notion.cli <명령어> [옵션]
```

| 명령어 | 설명 | 예시 |
|---|---|---|
| `add-stock` | 종목 등록 | `--name 삼성전자 --code 005930 --market 코스피 --sector IT --currency KRW` |
| `add-transaction` | 매매내역 등록 + 재계산 | `--code 005930 --type 매수 --qty 10 --price 70000 --fee 100 --date 2026-07-06` |
| `list-transactions` | 종목의 매매내역 조회(페이지ID 확인) | `--code 005930` |
| `edit-transaction` | 매매내역 수정(바꿀 항목만) | `--transaction-id <ID> --price 71000` |
| `delete-transaction` | 매매내역 삭제(휴지통) | `--transaction-id <ID>` |
| `add-dividend` | 배당금 등록 | `--code 005930 --date 2026-07-01 --pretax 5000 --posttax 4230` |
| `update-prices` | 전 종목 현재가 갱신 | (옵션 없음) |
| `sync-portfolio` | 포트폴리오 요약 재계산 | (옵션 없음) |
| `daily-update` | 현재가 갱신 + 재계산 (+KIS 대조) | (옵션 없음, 스케줄러용) |
| `kis-check-balance` | KIS 계좌 잔고와 Notion 대조(읽기 전용) | (옵션 없음) |

**시장구분 선택지**: 코스피 / 코스닥 / 나스닥 / NYSE / AMEX / 상해 / 심천 / 홍콩 / 기타
**통화 선택지**: KRW / USD / CNY / HKD
**매매구분**: 매수 / 매도

> `edit-transaction`/`delete-transaction`은 거래의 페이지ID가 필요합니다.
> 먼저 `list-transactions --code <종목코드>`로 ID를 확인하세요.

## 6. 자동 갱신 설정

매일 장 마감 후 자동으로 현재가·포트폴리오를 갱신하려면(수동으로 안 눌러도 되게):

- **Windows**: 작업 스케줄러에 등록 (README "Windows 작업 스케줄러" 절). 단, **PC가 켜져 있을 때만** 실행됩니다.
- **라즈베리파이/Docker**: `docker compose up -d`로 `scheduler` 컨테이너가 매일 16:00 자동 실행.
  24시간 켜져 있어 가장 확실합니다. → [라즈베리파이 설치 가이드](raspberry-pi-setup-guide.md)
- **리눅스 cron / APScheduler**: README "자동 스케줄링" 절 참고.

세 방식 모두 내부적으로는 `daily-update`를 실행합니다.

## 7. 증권사(KIS) 연동 — 선택

실제 한국투자증권 계좌 잔고와 내 Notion 기록이 일치하는지 대조할 수 있습니다(현재는 **읽기 전용**,
주문 실행 없음).

1. [KIS Developers](https://apiportal.koreainvestment.com/apiservice)에서 API 신청 → appkey/appsecret 발급
2. `.env`에 `KIS_APP_KEY`/`KIS_APP_SECRET`/`KIS_ACCOUNT_NO` 입력 (`KIS_IS_PAPER=true`는 모의투자)
3. `python -m stockapp_notion.cli kis-check-balance` 실행 → 잔고 불일치 항목을 알려줌
4. `daily-update`에도 자동 포함됨(설정된 경우에만)

> 모의투자로 충분히 검증한 뒤에만 `KIS_IS_PAPER=false`(실계좌)로 바꾸세요.
> 체결내역 자동 임포트 등 다음 단계는 개발 예정입니다.

## 8. 문제 해결

| 증상 | 확인/해결 |
|---|---|
| `환경변수 XXX가 설정되지 않았습니다` | `.env`에 해당 값이 있는지, 오타 없는지 확인 |
| Notion 401 unauthorized | `NOTION_TOKEN`이 맞는지, Integration이 페이지에 연결(Connect)됐는지 |
| `종목코드 XXX를 찾을 수 없습니다` | 매매/배당 입력 전에 `add-stock`(또는 웹 종목 등록)으로 먼저 등록 |
| 현재가가 0 | "현재가 갱신" 실행. 계속 실패하면 시장구분이 맞는지 확인(코스피↔코스닥 바뀌면 티커 틀림) |
| 해외종목 시세 실패 | 종목코드/시장 조합 확인(미국=코드 그대로, 상해=.SS, 심천=.SZ, 홍콩=.HK로 자동 변환됨) |
| 로그인 켰는데 앱이 시작 안 됨 | `FLASK_SECRET_KEY`를 `.env`에 채워야 함 |
| 중복 매매 경고 | 같은 조건 거래가 이미 있다는 알림일 뿐, 등록은 진행됨(분할매매 등 정상일 수 있음) |

**로그 확인**: 모든 실행 기록은 `logs/app.log`에 남습니다.
```bash
tail -50 logs/app.log        # 최근 50줄
grep ERROR logs/app.log       # 오류만
```
